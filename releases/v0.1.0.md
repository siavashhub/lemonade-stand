# v0.1.0 — Connect to your Lemonade server

_Release date: 2026-07-18_

This release makes a fresh install connect to Lemonade out of the box and lets
you configure the server connection from inside the app. It also adds optional
plan/todo list visualization so the agent can lay out and track multi-step work.

## Added
- **In-app connection settings.** Click the **server-status** pill in the top
  bar to set the Lemonade base URL and (optional) API key. The choice is saved
  and applied immediately — no `.env` edit or restart needed.
- **Agent planning tool.** The agent can now call a built-in `update_plan` tool
  to create or revise a short todo list for multi-step tasks. The plan appears as
  a live checklist in the transcript, updated in place as the model works through
  steps. Useful for providing structure and transparency on complex requests;
  automatically skipped for simple one-step work.
- **Context-usage indicator.** A compact pill in the top bar shows the current
  context-window usage as a percentage with a mini fill bar. Click the pill to
  open a donut breakdown popover (Option A design) that details category-wise
  token consumption (System, Tools, Messages, Results, Other, Reserved) and
  offers a **Compact Conversation** button to summarize older messages and
  reclaim context space. Helpful for monitoring prompt budget in long chats.

## Fixed
- **Packaged installs now find their bundled config.** The Pantry catalogue and
  configured MCP servers were empty on a fresh install because the app read
  `config/` from inside the read-only app bundle. Defaults are now seeded into a
  writable per-user directory on first run, so the Market is populated and your
  edits survive upgrades.
- **Correct default server port.** The app defaulted to `http://localhost:8000`,
  but Lemonade Server listens on **13305**. The default is now
  `http://localhost:13305/api/v1`, so the server-status indicator shows *online*
  without any configuration.
- **Speaker toggle now persists across restarts.** The spoken replies (TTS)
  on/off state is now saved to `config/settings.json` and restored on app
  startup, instead of resetting to enabled each time.
- **Stop spoken replies mid-playback.** While a TTS reply is playing, the
  speaker button turns into a stop control — click it to halt playback
  immediately if TTS was enabled by mistake. It reverts to the normal speaker
  toggle once playback finishes. Toggling the speaker off also silences any
  in-flight audio.

## Requirements
- Requires a running `lemond` server: (https://github.com/lemonade-sdk/lemonade)
- Local model with tool-calling support recommended (e.g. Omni, Qwen, Mistral)
  for best agent performance. Weaker models benefit from the planning tool to
  stay on track.
